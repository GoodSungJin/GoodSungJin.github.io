WebCafe Markup_03

<main> Markup

  • class="group1", class="group2", class="group3"

    • CSS 작업의 편의를 위해 각 영역을 나눠주었다.


class="group1" Markup

  • class="login", class="validation", class="term"

    • 눈에 보이는 순서로 Markup 해도 정보전달에 무리가 없다고 판단했다.


class="login" Markup

로그인섹션을 선형적인 구조로 그려보고 마크업했다.

  • <section>

    • 문서의 일반적인 구획을 나타내는 컨텐츠 그룹인 <section> 은 <h> 를 자식요소로 포함해야 한다.

  • <form>

    • <form>은 <input>이 있을 경우 사용하게 되는데, 데이터를 보내기 위함이다.
      또한, 같은 목적을 가진 위젯을 그룹화하는 <fieldset> 을 포함하고 <fieldset>의 라벨인 <legend> 를 같이 사용해야 한다.

  • <input>

    • <input> 과 <label>은 세트로 지정해줘야 한다.
      여기서 <label>인 아이디(text)를 클릭하면 <input>으로 바로 연결되는 효과를 얻을 수 있다.

  • name="text"

    • 서버로 데이터를 보낼 때 name 값으로 보낸다고 한다.

  • required

    • <input> 창을 필수로 입력하라는 텍스트를 보여준다.

  • placeholder="text"

    • <input> 창에 해당 영역의 역할을 텍스트로 보여준다.


class="vaildation" Markup

  • target="_blacnk"

    • <a> 태그 사용시 target="_blacnk" 속성을 사용하면 새창에서 열기가 가능하다.

  • title="text"

    • 텍스트에 마우스오버를 하면 텍스트를 설명해주는 툴팁이 열린다.


class="term" Markup

  • <dl>, <dt>, <dd>

    • <dl> 용어를 설명하는 목록을 만드는 태그이고, 자식요소로 용어의 제목인 <dt> 태그와, 용어를 설명하는 태그인 <dd> 를 두어야 한다.

  • alt="text"

    • 이미지를 사용하면 무조건 대체텍스트인 alt 속성을 줘야한다.

  • 새롭게 알게 된 사실

    • 입력창 옆에 글자를 클릭하면 자동으로 입력창으로 가는게 신기했는데 <label> 을 사용하면 가능하다는 것을 알았다.
    • 입력창 안에 글자는 CSS로 작업하는 줄 알았는데 <input> 의 속성인 required placeholder="text" 로 가능하다는 것을 알았다.
    • 필요에 따라 target="_blacnk" 속성을 사용해 새창으로 열기가 가능하다는 것을 알았다.
    • 용어를 설명하는 목록 태그인 <dl>, <dt>, <dd> 새롭게 알게되었다.
