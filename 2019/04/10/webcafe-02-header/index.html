WebCafe Markup_02

header


위에 헤더를 영역별로 구분한걸 토대로 그림을 선형적인 구조로 그려봤다.


  • Markup 순서

    1. h1
      - 사이트의 제목이 가장먼저 등장해야 된다고 생각했다.
    2. ul.member
      - 메뉴 영역으로 가기전 로그인, 회원가입등이 먼저 진행된다고 생각됐다.
    3. nav.navigation
      - 위에 2개를 하고 남은 걸 마지막으로 뒀다.

  • class="logo"

    • 제목을 나타내는 <h1> 태그를 사용했다. 또, 로고 이미지는 배경이 아니라 로고자체의 의미를 가지고 있는 것 같아 <img> 를 사용해 markup에 포함시키고, 링크사용을 위해 <a>를 사용했다.

  • class="member"

    • 순서에 상관없는 목록화를 위해 <ul> 을 사용하고, <a> 를 포함한 <li> 를 사용했다.

  • class="navigation"

    • 사이트의 주요 네비게이션 역할을 담는 태그인 <nav> 를 사용하고, 스크린리더 사용자를 위해 <h2>를 사용해 제목을 명시해줬다. 이때, 제목은 CSS를 이용해 시각적으로 보이지 않게 작업했다.
      메뉴는 <ul>,<li>을 사용했고, 키보드의 tab 접근을 위해 tabindex="0" 속성을 사용했다. 또한, 메뉴의 <li> 안에 서브메뉴 <ul>, <li>를 사용하고, <a>로 감싸줬다. 이떄, 서브메뉴의 제목은 시각적으로도 보이기 떄문에 서브메뉴 <ul> 의 자식으로 <h3>을 사용해 작업했다.


  • 새롭게 알게 된 사실

    1. 로고는 <img> 만 쓰고 그냥 넘겼는데 로고의 의미를 정확히 알고 <h1> 을 사용했다.
    2. 메인메뉴를 만들 때 <div> 를 사용했는데 <nav>를 사용했다.
    3. 눈으로는 안보이지만 <nav> 가 무엇인지 알려줘야 해서 <h2> 를 사용해 메인메뉴의 제목을 정의 해줬다.
